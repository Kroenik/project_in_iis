# ConVolunteer

ConVolunteer is a Streamlit app that helps volunteers create a profile and find
matching opportunities via a conversational agent.

## 1. Prerequisites

- Python 3.11+
- [uv](https://docs.astral.sh/uv/) (recommended)
- A Supabase project (Auth + Postgres database)
- An OpenAI API key

## 2. Clone and install dependencies

```bash
git clone <repo-url>
cd project_in_iis

uv sync
```

This will create a local virtual environment and install all dependencies from
`pyproject.toml`/`uv.lock`.

If you need to install `uv` first, see:
https://docs.astral.sh/uv/getting-started/installation/

## 3. Configure application secrets

Create a local secrets file at `.streamlit/secrets.toml`:

```toml
OPENAI_API_KEY = "your-openai-api-key"
SUPABASE_URL = "https://<your-project-ref>.supabase.co"
SUPABASE_KEY = "your-supabase-anon-or-publishable-key"
```

The app reads these values via Streamlit secrets, not from `.env`.

## 4. Prepare Supabase

### Enable authentication

In Supabase, enable Email/Password sign-in for your project.  
The app includes Sign Up and Sign In screens and uses Supabase Auth sessions.

### Create required tables

Run the following SQL in the Supabase SQL editor:

```sql
create table if not exists volunteer_profiles (
  id bigint generated by default as identity primary key,
  user_id uuid not null unique references auth.users(id) on delete cascade,
  name text,
  contact text,
  city text,
  zip_code int,
  availability jsonb default '{}'::jsonb,
  skills text[] default '{}',
  languages text[] default '{}',
  h_week int,
  start_date date,
  end_date date,
  recurring boolean,
  preference text,
  preference_embedding jsonb
);

create table if not exists opportunities (
  id bigint generated by default as identity primary key,
  org text,
  title text,
  summary text,
  tasks text[] default '{}',
  required_skills text[] default '{}',
  optional_skills text,
  language_requirements text[] default '{}',
  amount_volunteers int,
  schedule jsonb default '{}'::jsonb,
  hours_week int,
  recurring boolean,
  zip_code int,
  city text,
  email text,
  embedding jsonb
);
```

### Add RLS policies (recommended)

If Row Level Security is enabled, add policies so authenticated users can
access data:

```sql
alter table volunteer_profiles enable row level security;
alter table opportunities enable row level security;

create policy "profiles_select_own"
  on volunteer_profiles for select
  to authenticated
  using (auth.uid() = user_id);

create policy "profiles_insert_own"
  on volunteer_profiles for insert
  to authenticated
  with check (auth.uid() = user_id);

create policy "profiles_update_own"
  on volunteer_profiles for update
  to authenticated
  using (auth.uid() = user_id)
  with check (auth.uid() = user_id);

create policy "opportunities_select_all"
  on opportunities for select
  to authenticated
  using (true);

create policy "opportunities_insert_authenticated"
  on opportunities for insert
  to authenticated
  with check (true);
```

## 5. Run locally

```bash
uv run streamlit run frontend.py
```

Then open the local URL shown in the terminal (usually
`http://localhost:8501`).

## 6. Common issues

- `Required secret ... is not set`: check `.streamlit/secrets.toml`.
- Sign-in fails: verify Email/Password auth is enabled in Supabase.
- `schema mismatch` messages: check table names and column names in Supabase.
- Dependency issues after pulling changes: run `uv sync` again.

## Security note

Never commit real API keys or Supabase keys. If a key was ever committed,
rotate it immediately.
